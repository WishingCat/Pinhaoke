## JSON 结构

本科春季、暑期和秋季记录使用同一嵌套结构：

```json
{
  "课程类型": "专业课",
  "数据学期": "25-26学年第3学期",
  "详情链接": "https://elective.pku.edu.cn/...",
  "课程序号": "BZ2526301235260_14376",
  "基本信息": {
    "课程号": "01235260",
    "课程名": "3S野外综合实习",
    "课程类别": "任选、劳动教育课",
    "学分": "1.0",
    "教师": "李培军(教授),田原(副教授)",
    "班号": "1",
    "开课单位": "地球与空间科学学院",
    "专业": "",
    "年级": "2024",
    "上课时间及教室": "",
    "限数已选": "50 / 15",
    "自选PNP": "不可申请",
    "备注": ""
  },
  "详细信息": {
    "英文名称": "Practice on RS, GIS and GPS",
    "先修课程": "",
    "中文简介": "",
    "英文简介": "",
    "成绩记载方式": "等级制",
    "通识课所属系列": "",
    "授课语言": "中文",
    "教材": "",
    "参考书": "",
    "教学大纲": "",

```

春季本科拆成三个文件，课程类型由文件名决定；暑期本科和秋季本科在同一个文件内用 `课程类型` 标明来源 tab。英语课列表比普通类别多一个 `英语等级` 字段，如 `B` / `C` / `C+`，构建数据库时不会单独入列，但 JSON 会保留。

研究生 JSON 与本科来源不同，字段会由研究生建库脚本转换为研究生库 schema。后端统一把研究生课显示成 `研究生课`。秋季研究生详情页比春季研究生多保留了 `syllabus` 字段，后端会在详情 API 中一并返回。

## 数据库结构

五套数据库都提供：

- `basic_info`：课程列表字段
- `detail_info`：课程详情字段
- `courses_view`：列表和详情的连接视图
- `translations`：多语言译文，主键为 `(course_id, field, lang)`

本科春季、暑期和秋季 `basic_info` 额外带 `course_type`，研究生库没有原生 `course_type`，后端查询时固定映射为 `研究生课`。

`数据库构建脚本/build_common.py` 提供共享解析函数：

- `parse_schedule()`：把选课网拼接在一起的上课时间与教室拆成 `schedule` / `classroom` / `weekdays`
- `parse_first_period()`：提取最早节次，供 `time_asc` 排序使用
- `to_float()`：学分转换

## 注意事项

- 所有 JSON 源字段均按选课网文本保存，学分、限数/已选等字段在源 JSON 中仍是字符串。
- 同一物理课程可能被选课系统挂在多个类别下。后端 `/api/courses` 会按 `course_code + class_no + teacher` 合并列表卡片，并把 `course_type` / `category` 返回为数组。
- 课程详情页字段可能为空，数据库会保留空字符串。
- `限数已选` 是抓取时刻快照：春季数据来自 2026 年 3 月 20 日，暑期数据来自 2026 年 5 月 13 日，秋季数据来自 2026 年 7 月 9 日。
- 选课网抓取流程见 `北京大学选课网数据抓取/README.md`，不要用终端直接请求选课网页面详情接口。
