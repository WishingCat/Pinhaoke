# 课程与树洞评测数据说明

`课程数据/` 保存拼好课 V2 的北京大学选课系统源 JSON。源页面为 `https://elective.pku.edu.cn`，所有记录保留抓取时的课程列表字段和课程号详情字段。

## 学期与来源日期

| 页面学期 | 产品学期 | 范围 | 抓取日期 |
|---|---|---|---|
| `2025-2026学年第2学期` | 2026 春季 | 本科、研究生 | 2026-03-20 |
| `2025-2026学年第3学期` | 2026 暑期 | 本科 | 2026-05-13 |
| `2026-2027学年第1学期` | 2026 秋季 | 本科、研究生 | 2026-07-09 |

`限数已选`、教室、教师和备注等值是对应抓取日期的页面快照，不应解释为实时选课状态。

## API 合并后数量

选课网会把同一教学班挂在多个课程类型或类别下。`GET /api/courses` 按学段、课程号、班号和教师合并列表卡片，并保留完整 `course_type` / `category` 数组。因此页面数量小于 raw 行数：

| 学期 | API 卡片总数 |
|---|---:|
| 春季 | 3701 |
| 暑期 | 160 |
| 秋季 | 4421 |

春季和秋季的本科、研究生使用不同学段前缀，不能因为课程号相同而互相合并。API total 是产品验收口径；raw 行数是源数据与数据库构建口径，两者不可混用。

## 树洞课程评测快照

`数据库/树洞课程评测.db` 来自仓库外的 `树洞全量数据截止20260713` 快照。构建脚本扫描 2022-12 至 2026-07 的 44 个月度 JSON 分片，共 3561472 个帖子和 15245822 条已缓存回复。源 manifest 记录的回复缓存覆盖率为 95.24%，另有 761165 条回复未缓存，因此数据库只能筛选快照中实际存在的回复。

每个主题一定保留主帖，只保留包含课程体验、教学、作业、考核、给分、难度、考勤或明确推荐信息的回复。课程群、签到代办、考试时间询问、资料交易、求职招聘、升学考试与无评价信息的跟帖不进入数据库。课程名标签由五个课程库中的课程名和课程号匹配；没有可靠课程标签但具有明确评价内容的主题仍可通过全文搜索找到。

输出不保存 `authorTag`、`authorLabel`、身份字段或 `replyTo`。联系方式在入库前清除；公开字段仅包括树洞号、评论号、楼层、时间、来源月份、原帖链接、正文、课程标签和实体高亮区间。分类器版本、高亮版本、源数量和命中数量写入 `metadata`，用于健康检查和可复现验收。

高亮词典来自五个课程数据库。课程名只在条目已经关联该课程时标注；教师名必须与关联课程的任课教师一致，或紧邻“老师”“教授”等教师上下文。长名称优先于重叠短名称，区间互不重叠并使用 Unicode 码点偏移，确保中文、emoji 和历史控制字符不会造成前端错位。

## SQLite schema

五库都包含：

- `basic_info`：列表字段与解析后的 `schedule`、`classroom`、`weekdays`、`first_period`
- `detail_info`：课程号详情，与 `basic_info.id` 1:1
- `courses_view`：两表连接视图
- `translations`：自由文本译文，主键 `(course_id, field, lang)`

本科 `basic_info` 具有 `course_type` 和 `pnp`；研究生后端把课程类型合成为 `研究生课`，并把不存在的本科字段映射为空。研究生与本科详情列不完全相同，跨库 `UNION ALL` 必须输出相同列形状。

`数据库构建脚本/build_common.py` 提供：

- `parse_schedule()`：拆分选课网页面粘连的时间与教室
- `parse_first_period()`：提取最早节次供 `time_asc` 排序
- `to_float()`：转换学分

五个入口通过 `build_atomic.py` 在目标同目录构建临时库，严格校验源字段、冲突键、表与视图、外键、1:1 行数和完整性，通过后才替换正式文件，并同步目标父目录以确认目录项持久化。

树洞评测库包含：

- `metadata`：快照日期、分类器版本、源分片与命中数量、回复缓存覆盖率
- `threads`：主题级主帖、来源和相关回复计数
- `entries`：每个主题的主帖及筛选后的相关回复
- `thread_courses` / `entry_courses`：主题级和条目级课程标签
- `course_catalog`：搜索建议所需的课程名、课程号和命中数量
- `entity_aliases`：由全称同现关系提取并经复现、关键字与教师首字母校验的课程/教师缩写
- `entry_highlights`：正文中的课程/教师类型、`full` / `alias` 命中类型及起止码点偏移

`build_treehole_reviews.py` 使用目标同目录临时文件，完成 schema、行数、外键、高亮范围与重叠检查和 `PRAGMA integrity_check` 校验后才原子替换评测库，并同步父目录。`--enrich-existing` 可在不重扫树洞分片的情况下原子刷新高亮。源树洞快照不纳入仓库；正式评测库使用 Git LFS 保存。

## 翻译口径与缺口

仓库数据库快照（2026-07-11）：

| 数据库 | translations 行数 |
|---|---:|
| 春季本科 | 100156 |
| 春季研究生 | 39445 |
| 暑期本科 | 10010 |
| 秋季本科 | 159138 |
| 秋季研究生 | 52430 |

表行数不是完整覆盖率。一个课程可以有多个字段，每个字段最多七种目标语言；空源字段不需要译文，非空字段也可能仍缺部分语言。春季本科的 `syllabus`、`evaluation`、`reference_book` 和春季研究生 `reference_book` 是需要按 pending 扫描复核的长字段。

后端仅用非空译文替换原文，缺失译文回退中文。重建任一数据库会把 `translations` 重建为空表；备份和补译规则见 [翻译 README](../北京大学课程数据翻译/README.md)。

## 数据保护与验收

- JSON 保留选课网文本；源文件中的学分和限数/已选仍可能是字符串。
- 详情字段为空时保留空字符串，不推测缺失内容。
- 不直接用终端请求选课网详情接口；使用 [已登录 Chrome 页面内流程](../北京大学选课网数据抓取/README.md)。
- 普通程序修复不得改写正式 JSON 或数据库；构建测试使用临时副本。
- 树洞评测更新不得加入作者身份、回复关系或未清除的联系方式。
- 数据验收同时检查 raw 行数、API total、评测命中数、`PRAGMA foreign_key_check`、`PRAGMA integrity_check` 和数据库 SHA-256。
