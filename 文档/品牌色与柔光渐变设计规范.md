# 拼好课品牌色与柔光渐变设计规范

> 整理自 2026-07 的首页视觉改版。所有颜色源自 Pin logo 的两端色，延展成一套完整的青绿色系。
> 对应实现全部在 `index.html` 的 CSS 中，本文给出色值、命名、用途与可复用的设计公式。

## 一、核心色系：薄荷绿 → 深青（Mint → Deep Teal）

整站只有一个主色相：**青绿（Hue ≈ 170°）**。所有渐变都是这个色相的亮暗两端连线，
从不跨色相渐变——这是整套设计看起来"干净"的关键。

| 色值 | 常用名 | 角色 |
|---|---|---|
| `#5CE8C6` | 亮薄荷 Bright Mint | 深色模式渐变亮端 |
| `#35C6A7` | 加勒比海绿 Caribbean Green | logo 渐变亮端 |
| `#2FBE9E` | 薄荷绿 Mint | 标题/按钮渐变亮端 |
| `#167B72` | 深青 Deep Teal / Pine | 渐变暗端（logo、标题共用） |
| `#14606A` | 石油蓝 Petrol | 标题渐变最深端，收尾用 |
| `#233F48` | 墨青 Dark Slate | logo 里 "in" 两个字母的颜色 |
| `#F6FFFC` | 薄荷白 Mint White | logo 底色（看起来是白，实际带 2% 薄荷） |

### 全站语义色变量（浅色 / 深色模式）

```css
/* 浅色 */
--accent:      #0F8A7D;   /* 交互主色：链接、选中态、focus 边框 */
--accent-deep: #0C6F65;   /* hover 加深 */
--accent-soft: #C9EEE7;   /* 选中项底色 */
--accent-mist: #E8F7F3;   /* focus 光环、最浅的衬底 */

/* 深色（同色相整体提亮 + 转半透明） */
--accent:      #46D6B8;
--accent-deep: #6BE3CA;
--accent-soft: rgba(70, 214, 184, 0.28);
--accent-mist: rgba(70, 214, 184, 0.13);
```

## 二、线性渐变（Linear Gradients）

### 1. 标题渐变文字（首页"拼好课"）

```css
h1 {
  background-image: linear-gradient(118deg, #2FBE9E 8%, #167B72 58%, #14606A 94%);
  -webkit-background-clip: text;
  background-clip: text;
  color: transparent;
}
/* 深色模式提亮一档 */
html[data-theme="dark"] h1 {
  background-image: linear-gradient(118deg, #5CE8C6 8%, #35C6A7 58%, #27A68E 94%);
}
```

要点：
- **118° 斜角**——左上亮、右下深，比水平/垂直渐变更有动感，比 45° 更稳重；
- 三段色标（8% / 58% / 94%）让中间的主体色占最大面积，两端只做提亮和收深；
- 深色模式不是换色，是**同色相整体提亮**，渐变关系不变。

### 2. logo / 按钮 / 头像渐变

```css
/* logo 的 P（SVG linearGradient，斜向） */
#35C6A7 → #167B72

/* 学期切换选中胶囊 */
background: linear-gradient(135deg, #2FBE9E, #167B72);           /* 浅色 */
background: linear-gradient(135deg, #4FE0BE, #2BB593);           /* 深色 + 深色文字 */

/* 弹窗章节标题的竖条 */
background: linear-gradient(180deg, #2FBE9E, #167B72);
```

小面积元素用 **135°（或垂直 180°）两段式**即可，不需要三段色标。

### 3. 关于卡片的头像圆片（多色但同构）

三组鸣谢头像刻意换了色相（取自课程类型徽章色，Tailwind 命名）：

```css
靛蓝 Indigo: linear-gradient(135deg, #6366F1, #4F46E5)
琥珀 Amber:  linear-gradient(135deg, #F59E0B, #D97706)
玫红 Rose:   linear-gradient(135deg, #F43F5E, #E11D48)
```

规则一致：**同色相亮→暗、135° 斜角**，所以颜色不同依然像一家人。

## 三、柔光光晕（Aurora / Mesh Glow）

首页标题背后那层"抓不到边界"的柔光，是本次设计的签名元素。

```css
.hero { position: relative; }

.hero::before {
  content: '';
  position: absolute;
  top: -30px; left: 50%;
  width: min(720px, 92vw); height: 250px;
  transform: translateX(-50%);
  background:
    /* 主光晕：品牌薄荷绿，偏左 */
    radial-gradient(45% 60% at 33% 42%, rgba(53, 198, 167, 0.17), transparent 70%),
    /* 副光晕：一点靛蓝作对比温度，偏右 */
    radial-gradient(40% 55% at 67% 46%, rgba(79, 70, 229, 0.10), transparent 70%);
  filter: blur(8px);
  pointer-events: none;   /* 别挡住下面的交互 */
}

html[data-theme="dark"] .hero::before {
  background:
    radial-gradient(45% 60% at 33% 42%, rgba(53, 198, 167, 0.12), transparent 70%),
    radial-gradient(40% 55% at 67% 46%, rgba(115, 150, 242, 0.10), transparent 70%);
}
```

### 柔光配方（可直接套用）

1. **两团光，不要一团**：主色一团 + 对比色一团，错开摆放（33% / 67%），单团会显得呆板；
2. **透明度控制在 0.10 ~ 0.17**：浅色底再高就"脏"了；深色底还要再降（0.10 ~ 0.12）；
3. **radial-gradient 到 70% 处收干净**（`transparent 70%`），避免光晕边缘出现硬边；
4. **叠一层 `blur(8px)`**：径向渐变本身有方向感，模糊之后才是"光"而不是"色块"；
5. **伪元素承载 + `pointer-events: none`**：不侵入 DOM，也不挡点击；
6. 宽度用 `min(720px, 92vw)` 自适应，手机上不会溢出。

### 类似手法的小面积应用

```css
/* 关于卡片页眉的渐变衬底：线性版的"薄光" */
.about-head {
  background: linear-gradient(120deg, rgba(53, 198, 167, 0.16), rgba(22, 123, 114, 0.05));
}
```

同一思路的降级版：把透明度压到 0.05~0.16 的线性渐变，用作卡片页眉/分区的底色。

### 全站环境光（page ambient aurora）

在 `body::before` 上铺一层 `position: fixed`、`z-index: -1` 的极淡三团光晕，跟随视口固定、
缓慢漂移，让整页始终有一层"空气感"，又不抢任何内容焦点。

```css
/* body 必须透明，页面底色交给 html 承载，负 z-index 的光才透得出来 */
body::before {
  content: '';
  position: fixed;
  inset: -10% -10% auto -10%;
  height: 620px;
  z-index: -1;
  pointer-events: none;
  background:
    radial-gradient(38% 60% at 15% 0%,  rgba(53, 198, 167, 0.10), transparent 62%),
    radial-gradient(34% 56% at 88% 4%,  rgba(79, 70, 229, 0.07), transparent 60%),
    radial-gradient(30% 50% at 60% 30%, rgba(20, 96, 106, 0.05), transparent 66%);
  filter: blur(6px);
  animation: aurora-drift 26s ease-in-out infinite alternate;
}
@keyframes aurora-drift {
  from { transform: translate3d(0, 0, 0)    scale(1);    }
  to   { transform: translate3d(0, 14px, 0) scale(1.05); }
}
```

要点：
- **三团比两团更松弛**：环境光铺满顶部宽幅，用三团错落（15% / 88% / 60%）比签名柔光的两团更自然；
- **透明度更低（0.05 ~ 0.10）**：它是背景的背景，比标题柔光还要淡一档；
- **`animation … alternate` 慢漂移**：26s 一个来回、位移仅 14px + 缩放 5%，慢到几乎察觉不到，只留"活着"的感觉；
- **`prefers-reduced-motion` 下直接 `animation: none`**：0.001s 时长会让交替动画逐帧闪烁，必须关掉而不是提速；
- 承载它的 `body` 要透明、底色移交 `html`，否则不透明背景会盖住负层。

### 发丝渐变分隔线（gradient hairline）

比纯色 `border` 更精致：中间品牌色最亮、两端淡出，用在顶栏底边和页脚上沿。

```css
/* 页脚上沿：居中、两端淡出 */
.site-footer::before {
  content: '';
  position: absolute; top: 0; left: 50%;
  transform: translateX(-50%);
  width: min(560px, 90%); height: 1px;
  background: linear-gradient(90deg, transparent,
    color-mix(in srgb, var(--accent) 55%, transparent) 50%, transparent);
}
```

`color-mix(in srgb, var(--accent) 55%, transparent)` 让分隔线自动跟随浅/深色主色，无需写两套。

### 交互态里的柔光

同一套光晕思路也用在悬停/加载态上，保持全站语言统一：

```css
/* 课程卡右上角光晕：常态极淡，悬停也只加一档 */
.course-card::after { opacity: 0.045; }         /* 深色 0.07 */
.course-card:hover::after { opacity: 0.09; }    /* 深色 0.12 */

/* "加载更多"悬停：线性薄光而非纯色填充 */
.load-more:hover {
  background: linear-gradient(120deg, rgba(53,198,167,0.10), rgba(53,198,167,0.02) 65%), var(--surface);
}

/* 加载指示器：锥形渐变 + mask 抠出圆环，颜色沿圆周淡入 */
.spinner {
  background: conic-gradient(from 90deg, transparent 8%, var(--accent));
  -webkit-mask: radial-gradient(farthest-side, transparent calc(100% - 3.5px), #000 calc(100% - 3px));
}
```

原则：**常态克制、交互轻点**。卡片角光常态压到 0.045，悬停也只到 0.09——柔光是氛围，不是信息，
宁可"差一点看不见"，也不要糊成一团色块。

## 四、总结：三条可复用公式

1. **渐变公式**：同一色相的亮端 → 暗端，118°~135° 斜角；大面积三段色标，小面积两段即可。
2. **柔光公式**：主色 + 对比色两团（大面积可三团）径向光晕错开放置，透明度 ≤ 0.17（环境光 ≤ 0.10、卡片角光 ≤ 0.05），`transparent ~70%` 收边，再加 blur。
3. **深色模式公式**：不换色相，整体提亮一档；半透明色则降低不透明度。
4. **发丝公式**：需要分隔线时用 `linear-gradient(90deg, transparent, color-mix(accent 55%), transparent)` 代替纯色 border，中亮两端淡。
