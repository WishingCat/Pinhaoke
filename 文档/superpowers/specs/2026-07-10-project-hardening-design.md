# 拼好课项目全面修复与文档收敛设计

日期：2026-07-10

## 目标

在不改写现有课程 ID、不重新调用付费翻译 API、不拆分整个单页应用的前提下，修复本次全仓审计中已经复现的课程完整性、筛选排序、详情展示、抓取建库、翻译脚本、部署安全和前端可访问性问题，并把项目说明收敛到少数权威文档。

完成后应满足：

1. 五个 SQLite 数据库内容保持完整，现有课程 URL 继续可用。
2. 本科与研究生课程不会互相误合并；同一学段内的重复挂载仍合并为一张卡片。
3. 搜索、筛选、排序、分页和详情在三学期与八种界面语言下行为稳定。
4. 抓取或建库失败不会覆盖正式 JSON、数据库或译文。
5. 生产代码及部署文件不再由 Web 服务用户持有，部署失败必须返回失败状态。
6. 文档只有唯一事实来源，不保留已经完成且内容重复的计划与设计副本。
7. 清理本机 Git 中 15,240 个不可达对象，并验证仓库、LFS 和数据库仍可恢复。

## 方案选择

采用聚焦式全面修复：修复所有有复现证据的核心问题，同时维持当前 FastAPI、单文件前端和 SQLite 架构。

不采用最小补丁方案，因为它会继续留下抓取覆盖正式数据、部署权限和无自动测试等已知风险；不采用结构重写方案，因为拆分 `app.py`、`index.html`、更换数据库身份体系或改成完整 release 平台会扩大上线风险，且不是本轮必要条件。

## 后端与 API

### 数据库连接

- `get_db()` 默认学期改为 `fall`，与 API 和前端一致。
- 数据库通过 SQLite URI `mode=ro` 只读打开，并设置 `query_only`。
- 连接创建后立即进入 `try/finally`；即使 `ATTACH` 失败也必须关闭主连接。
- 静态目录和首页使用 `BASE_DIR` 绝对路径，使应用可以从任意工作目录导入和启动。

### 列表合并

列表源数据先生成稳定分组键：`学段前缀 + course_code + class_no + teacher_key`。教师为空时以课程 ID 作为唯一 `teacher_key`。

筛选采用“两阶段”语义：

1. 在完整源数据中找出满足搜索或筛选条件的分组键。
2. 回到完整源数据聚合这些分组，从而保留该课程的全部课程类型、类别和非空字段。

这会避免筛选某一类别后卡片 ID、专业字段和徽章集合发生变化。本科与研究生使用不同学段前缀，因此春季恢复为 3701 张卡、秋季恢复为 4421 张卡；暑期保持 160 张合并卡。

同组课程按详情非空字段数量选择代表 ID；分数相同时选择本地整数 ID 较小者。这样课程卡优先打开信息更完整的重复记录，同时旧 ID 详情链接仍然有效。

### 搜索与排序

- 非中文请求在列表源查询中联接目标语言的 `course_name`、`classroom` 和 `notes` 译文。
- 搜索同时匹配原课程名、显示语言课程名、英文名、教师、原教室、显示语言教室和课程号。
- 前端将“拼音/A-Z”改为“课程名正序/倒序”；后端新增规范的名称排序值，同时继续接受旧 `pinyin` / `pinyin_desc` 参数以兼容共享链接。
- 名称排序按当前 API 返回的显示名称进行；中文不再宣称为真正拼音排序。
- 所有排序最后追加唯一 `id`；随机排序在聚合结果外层按代表 ID 计算哈希并追加 `id`，保证同一种子跨页稳定。

### 输入和详情

- 非法学分返回 422，不再造成 SQL 参数数量错误。
- `lang`、`weekday`、`term` 和排序值使用允许列表；页码设置合理上限并在越界时返回空列表。
- 课程 ID 必须匹配 `^[ugsar][1-9][0-9]*$`，拒绝 `u01`、`u+1` 等别名。
- 详情 API 增加 `textbook` 和 `reference_book`，并允许对应译文覆盖。
- 空白译文使用 `strip()` 判断并回退原文，避免空白字符串覆盖有效课程内容。
- 增加轻量健康检查，确认五库可只读打开、关键表存在且基本信息与详情行数一致。

## 前端

### 请求与状态

- `loadMore()` 在请求进行时直接拒绝重复触发；请求页码只在成功后提交，失败不跳页。
- 加载按钮在请求期间禁用，并在成功、失败和中止后恢复正确状态。
- 复制课程链接前同步 URL，复制当前完整 `location.href`，保留学期、语言、筛选、排序、随机种子和课程弹窗。

### 详情和多语言

- 展示教材与参考书。
- 非中文界面把数据库译文标为“翻译简介”，原始英文简介标为“英文简介”，避免把德语、日语等内容标成中文简介。
- 课表本地化补齐词语边界，并翻译已知附加短语；无法识别的原文保留，不进行破坏性猜测。
- 缺失的有限枚举继续回退中文；本轮补齐审计确认的课程类型、院系、类别和职称词典缺口。

### 可访问性与窄屏

- 自定义筛选提供可访问名称、`listbox/option` 语义、展开状态、方向键、Enter、Escape 和焦点移动。
- 详情弹窗使用标题关联，打开后聚焦弹窗、限制焦点在弹窗内，背景设为 `inert`，关闭后恢复触发元素。
- 复制按钮始终保留可访问名称。
- 修复 320px 视口横向溢出；学期切换可在自身区域适配而不撑宽页面。
- 加深浅色模式中过低对比度的正文、徽章和辅助文本颜色，不改变品牌主色结构。
- 删除已经返回 404 的 HarmonyOS 字体请求，使用稳定的系统字体回退。
- 百度统计关闭自动携带查询串的 pageview，只上报不含搜索词和筛选参数的路径。

## 抓取与建库

### 抓取接收器

三个接收器复用一个公共实现，但保留原有命令入口。启动时生成一次性随机令牌，并打印包含令牌的注入命令。

接收器只接受：

- 来源为 `https://elective.pku.edu.cn` 的请求；
- 携带正确一次性令牌的 `/inpage.js`、`/progress` 和 `/done`；
- 小于固定上限的 JSON 请求体；
- 与目标学期一致、`rows` 非空、`errors` 为空且关键 validation 项通过的 payload。

原始 payload 先原子写入临时目录。只有通过校验的数据才原子替换正式课程 JSON。拒绝的数据保留在临时目录并返回非 2xx，接收器不发布正式文件。

### 页面内脚本

- 保持只请求列表、分页和 `goNested.do` 详情；不得出现 `addToPlan.do` 或加入选课计划调用。
- 开始前确认页面文本包含目标学期。
- 本科抓取任一类别失败时不发布部分数据；系统提示或登录失效立即全局停止。
- 详情缓存提升为整个抓取任务共享，避免跨类别重复请求同一 `课程序号`。
- 网络请求加入超时、有限重试和退避；系统反爬提示不重试。
- 开课单位继续提交 `ALL`，确保带选项的类别不会停留在社会学院默认值。

### 数据库构建

五个建库入口都先读取并验证 JSON，再在目标目录创建临时 SQLite：

1. 严格检查必要字段、学分格式和唯一键；发现冲突即失败并报告，不再静默跳过。
2. 创建 `basic_info`、`detail_info`、`translations`、索引和 `courses_view`。
3. 校验预期行数、1:1 详情、外键和 `PRAGMA integrity_check`。
4. 全部通过后关闭连接并原子替换正式数据库。

本轮不自动迁移或恢复译文；正式数据库只有在用户明确运行建库命令时才会被替换，文档继续要求重建前备份译文。

## 翻译脚本

- API key 改为真正发起翻译时才检查，因此三个脚本在无密钥环境下可以导入和显示 `--help`。
- `certifi` 使用可选导入和系统证书回退，不增加生产依赖。
- `--only` / `--db` 只初始化和扫描选中的数据库，不再修改其他库。
- 空白译文视为缺失；API 返回空白文本时拒绝写入。
- API 调用成功后，数据库写入单独重试，数据库锁不能触发重复付费 API 调用。
- 任何未处理失败都以非零状态退出。
- 补齐春季本科 `syllabus` / `evaluation` / `reference_book` 和春季研究生 `reference_book` 的兜底任务矩阵，但本轮不调用翻译服务。

## 依赖与部署

- 将 FastAPI、Uvicorn 和 Starlette 更新到经 `pip-audit` 验证无已知漏洞的兼容版本，并在新临时环境中运行完整测试。
- 部署脚本使用 `flock` 防止并发执行；先完成 Git、LFS 和依赖预检，再停止服务。
- Git LFS 可用性在 reset 之前检查，目标提交的 LFS 对象先行下载。
- requirements 使用内容哈希判断是否需要重建环境，并在启动前运行 `pip check`。
- 删除把整个仓库递归交给 `www-data` 的行为；代码、Git、虚拟环境和数据库由 root 持有，服务用户只读。
- systemd 增加无新权限、只读文件系统、私有临时目录、空 capability 和严格 umask 等沙箱设置。
- HTTP 烟测失败必须非零退出；烟测覆盖健康检查、秋季筛选与课程列表，不只检查首页文件。
- 仓库 Nginx 配置与生产 HTTPS/Certbot 结构对齐，去掉重复 MIME 警告、固定文件名的 `immutable`，增加必要安全头；安装前必须 `nginx -t`。
- 新增 `deploy/README.md`，记录证书路径、更新流程、回滚、日志、权限和烟测命令。

## Git 清理

清理前再次验证：

- 工作树干净；
- 本地 HEAD 与远端 `main` 一致；
- 两个秋季 LFS 指针、工作文件哈希和远端对象一致；
- 五库均通过完整性检查。

随后安装本机 Git LFS，执行仅针对不可达对象的清理与仓库压缩。清理后重新运行 `git fsck`、`git status`、LFS 哈希、数据库完整性和磁盘占用检查。预计删除 15,240 个不可达对象并释放约 124.23 GiB，不删除任何分支、标签或 reflog。

今后翻译或建库在仓库外的临时数据库中进行，只把最终数据库一次性原子导入仓库，避免 Codex 检查点为每次 SQLite 写入保存完整二进制快照。

## 文档收敛

最终保留的权威说明：

- `README.md`：产品、运行入口、功能、项目结构和文档索引。
- `AGENTS.md`：工程架构、API 契约、关键约定和验证方式。
- `CLAUDE.md`：只指向 `AGENTS.md`，不复制正文。
- `北京大学选课网数据抓取/README.md`：抓取、校验与建库操作。
- `北京大学课程数据翻译/README.md`：翻译任务、参数、覆盖率和验收。
- `课程数据/数据说明.md`：数据来源、时间、记录数、schema 和已知缺口。
- `deploy/README.md`：生产部署与运维。
- `归档/README.md`：解释仍保留的旧文件，防止误用。

删除已完成且被权威 README 覆盖的暑期计划、暑期设计和品牌色说明。品牌 CSS 以 `index.html` 为唯一事实来源。访问统计 PDF 作为历史制品保留，不作为现行说明文档。

本设计和后续实施计划仅在开发过程中保留；项目完成并将有效信息并入上述权威文档后删除，避免再次形成过期说明。

## 测试与验收

新增标准库 `unittest` 测试，避免为了测试扩大生产依赖。至少覆盖：

- 三学期卡片总数和本研分离；
- 筛选前后代表 ID、全部徽章和非空字段稳定；
- 随机排序同种子稳定、跨页无重复遗漏；
- 目标语言课程名搜索和显示名称排序；
- 非法学分、星期、语言、页码和课程 ID；
- 教材、参考书和空白译文回退；
- 任意工作目录导入应用；
- 原子建库失败时正式数据库不变；
- 五种建库 schema 都有空 `translations` 表；
- 接收器令牌、来源、请求体、学期、错误和 validation 校验；
- 三个翻译脚本无密钥 `--help`、空白译文和失败退出状态。

前端使用真实浏览器验证桌面与 320px/390px 手机视口，覆盖重复加载、URL 复制、筛选键盘操作、弹窗焦点、横向溢出、浅深色对比和新增详情字段。最终再执行 Python 编译、JavaScript 语法、Ruff、ShellCheck、数据库 integrity、`pip-audit`、本地 API 冒烟和线上只读基线对比。

## 边界

本轮不做：

- 不重新抓取选课网，不触碰加入选课计划。
- 不调用 DeepSeek 或其他付费翻译 API。
- 不重建或改写当前五个正式数据库。
- 不迁移现有 URL 到新的稳定 ID 体系。
- 不把 `app.py` 或 `index.html` 拆成新框架。
- 不主动 push、部署或修改生产服务器；完成本地修复和验证后另行确认外部发布操作。
